## 1. Введение

**RK Web Monitor** — это современная веб-панель для мониторинга продаж в ресторанах, работающих на кассовой системе R-Keeper 7. Приложение заменяет устаревший WebMonitor 4.11 (ASP.NET + dojo mobile) на актуальный стек Next.js 16 + TypeScript + Tailwind CSS.

### Что умеет программа

- **Дашборд** — сводка смены: выручка, чеки, средний чек, активные столы, графики
- **Баланс** — отчёт по типам оплат (наличные, карты, безнал) с долями
- **Выручка** — детализация по валютам и способам оплат
- **Чеки** — список с поиском + drill-down до позиций и скидок
- **Заказы** — распределение столов по официантам
- **Расход блюд** — топ-10 + полная таблица с сортировкой и поиском
- **Официанты / Кассиры / Станции** — выручка с детализацией по валютам
- **Планы залов** — интерактивная SVG-карта столов с цветовой индикацией
- **Кассовая дата** — текущая дата и период смены
- **Сервис-печать** — конфигурация принтеров
- **Сообщения** — отправка сообщений персоналу
- **Настройки** — управление серверами RK7 и пользователями системы

### Системные требования

**Для конечного пользователя:**
- Windows 10/11 x64 или Windows Server 2019+
- 2 ГБ оперативной памяти
- 500 МБ свободного места на диске
- Современный браузер (Chrome, Edge, Firefox, Safari)
- Права администратора (для установки)
- Сетевой доступ к серверу R-Keeper 7 (если используете живое подключение)

**Node.js НЕ требуется** — установщик включает portable Node.js.

## 2. Установка на Windows

### Шаг 1. Получение установщика

Скачайте файл `RKWebMonitor-2.0.0-setup.exe` со страницы [Releases](https://github.com/AVzhirov/webmon/releases) репозитория.

### Шаг 2. Запуск установщика

1. Найдите скачанный файл `RKWebMonitor-2.0.0-setup.exe`
2. **Правый клик → "Запуск от имени администратора"**
3. Если Windows SmartScreen блокирует запуск:
   - Нажмите **"Дополнительные сведения"**
   - Нажмите **"Выполнить в любом случае"**
4. Разрешите приложению внести изменения (UAC)

### Шаг 3. Мастер установки

| Экран | Что выбрать |
|-------|------------|
| Приветствие | Нажмите "Далее" |
| Лицензия | Прочитайте MIT-лицензию, нажмите "Принимаю" |
| Папка установки | По умолчанию `C:\Program Files\RK Web Monitor` — оставьте как есть |
| Дополнительные задачи | Рекомендуется выбрать все: иконка рабочего стола, автозапуск, правило Firewall |
| Готово к установке | Нажмите "Установить" |
| Завершение | Оставьте галочку "Запустить RK Web Monitor" |

### Шаг 4. Первый запуск

После установки автоматически:
1. Инициализируется база данных SQLite
2. Создаётся правило Windows Firewall для порта 8083
3. Запускается Windows-служба `RKWebMonitor` через NSSM
4. Через 5 секунд открывается браузер на `http://localhost:8083`

Если браузер не открылся автоматически — откройте вручную адрес `http://localhost:8083`.

### Шаг 4. Управление Windows-службой

Приложение устанавливается как **Windows Service** с именем `RKWebMonitor` через NSSM. Это означает:

- ✅ **Автозапуск** при старте Windows (если выбран при установке)
- ✅ **Работа от имени SYSTEM** — не нужно входить в систему пользователю
- ✅ **Авто-перезапуск** при сбое (через 5 секунд)
- ✅ **Логирование** в `logs\server.log` с ротацией 10 МБ

**Управление через меню Пуск:**

| Ярлык | Что делает |
|-------|-----------|
| Открыть в браузере | Открывает http://localhost:8083 |
| Запустить службу | `start.bat` — запуск службы |
| Остановить службу | `stop.bat` — корректная остановка |
| Перезапустить службу | `restart.bat` — для применения настроек порта |
| Удалить программу | Деинсталляция |

**Управление через командную строку (от администратора):**

```cmd
sc start RKWebMonitor       :: запуск
sc stop RKWebMonitor        :: остановка
sc query RKWebMonitor       :: статус
sc config RKWebMonitor start= auto   :: автозапуск
sc config RKWebMonitor start= demand :: ручной запуск
```

**Управление через `services.msc`:**

1. Win+R → `services.msc` → Enter
2. Найти службу "RK Web Monitor"
3. Правый клик → Запуск / Остановить / Перезапустить / Свойства

### Шаг 5. Смена порта

По умолчанию приложение работает на порту **8083** (как в оригинальном WebMonitor 4.11). Чтобы изменить порт:

1. Войдите как администратор
2. Откройте **Настройки** → вкладка **"Система"**
3. В поле "Порт" укажите новое значение (1024–65535)
4. При необходимости выберите "Хост" (0.0.0.0 — все интерфейсы, 127.0.0.1 — только localhost)
5. Нажмите **"Сохранить настройки"**
6. Нажмите **"Перезапустить службу"** (или запустите `restart.bat`)
7. Дождитесь перезапуска (2-5 секунд)
8. Обновите правило Windows Firewall, если порт изменился

## 3. Первый запуск и настройка

### Шаг 1. Вход в систему

По умолчанию создан один пользователь:

| Поле | Значение |
|------|---------|
| Логин | `admin` |
| Пароль | `admin` |
| Роль | Администратор |

⚠️ **ВАЖНО:** Смените пароль администратора сразу после первого входа!

### Шаг 2. Выбор сервера

На экране авторизации выберите сервер RK7 из списка. По умолчанию созданы 2 демо-сервера:

- **Обучение (демо)** — работает с тестовыми XML-отчётами (47 файлов в комплекте)
- **Test (демо)** — резервный демо-сервер

Для реальной работы нужно добавить свой сервер — см. [раздел 4](#4-управление-серверами-rk7).

### Шаг 3. Открытие настроек

После входа нажмите кнопку **"Настройки"** (шестерёнка в правом верхнем углу) или перейдите в раздел **"Настройки"** в боковом меню.

## 4. Управление серверами RK7

### 4.1. Добавление нового сервера

1. Откройте **Настройки** → вкладка **"Серверы"**
2. Нажмите кнопку **"Добавить"**
3. Заполните форму:

#### Поле "Название"
Произвольное имя для отображения в списке. Например: "Главный ресторан" или "Питерский филиал".

#### Поле "Тип сервера"

| Тип | Описание | Пример адреса |
|-----|----------|--------------|
| 📊 **Демо-XML** | Папка с XML-отчётами для тестов | `public/demo-data/xml` или `C:\rk7\reports` |
| 🌐 **RK7 TCP** | Прямое TCP-соединение с сервером RK7 | `192.168.1.10:15551` |
| 🔗 **RK7 HTTP** | REST-эндпоинт XML-шлюза RK7 | `http://192.168.1.10/rk7/api` |

#### Поле "Адрес"
- Для **демо-XML**: путь к папке (относительно приложения или абсолютный)
- Для **RK7 TCP**: `IP:порт` (например, `192.168.1.10:15551`)
- Для **RK7 HTTP**: полный URL (например, `http://server/rk7`)

#### Поля для TCP/HTTP-серверов (показываются при выборе типа)

- **Логин RK7** — имя пользователя R-Keeper (обычно "manager")
- **Пароль RK7** — пароль пользователя R-Keeper
- **CryptKey** — ключ шифрования (если используется в вашей конфигурации RK7)

#### Чекбоксы

- **Включён** — сервер активен и виден на экране авторизации
- **По умолчанию** — будет автоматически выбран при входе

### 4.2. Проверка соединения

После добавления сервера нажмите иконку **Wi-Fi** рядом с ним:

- 🟢 Зелёная галочка — соединение установлено
- 🔴 Красный крестик — ошибка (проверьте адрес, логин, пароль, firewall)

Для демо-XML проверяется существование папки и количество XML-файлов.
Для TCP — пытаемся установить TCP-соединение с IP:порт.
Для HTTP — делаем HEAD-запрос.

### 4.3. Редактирование сервера

1. Нажмите иконку **карандаша** рядом с сервером
2. Измените нужные поля
3. Пароль обновляется только если введён новый (оставьте пустым — не менять)
4. Нажмите **"Сохранить"**

### 4.4. Удаление сервера

1. Нажмите иконку **корзины** рядом с сервером
2. Подтвердите удаление
3. ⚠️ **Нельзя удалить последний сервер** — должен остаться хотя бы один

### 4.5. Сервер по умолчанию

Чтобы сделать сервер выбранным автоматически при входе:

1. Нажмите иконку **короны** рядом с сервером
2. Предыдущий сервер по умолчанию потеряет этот статус

## 5. Управление пользователями

### 5.1. Роли пользователей

| Роль | Права |
|------|-------|
| 🔴 **Администратор** | Полный доступ: все отчёты + управление серверами и пользователями + настройки |
| 🔵 **Менеджер** | Просмотр всех отчётов + отправка сообщений. Без доступа к настройкам |
| ⚪ **Просмотр** | Только чтение отчётов. Без отправки сообщений и настроек |

### 5.2. Создание пользователя

1. Откройте **Настройки** → вкладка **"Пользователи"**
2. Нажмите **"Создать"**
3. Заполните форму:

- **Логин** — уникальное имя пользователя (например, `manager1`)
- **Отображаемое имя** — как имя будет показано в интерфейсе (например, "Иван Иванов")
- **Роль** — Администратор / Менеджер / Просмотр
- **Статус** — Активен (может входить) или заблокирован
- **Пароль** — минимум 4 символа

4. Нажмите **"Сохранить"**

### 5.3. Редактирование пользователя

1. Нажмите иконку **карандаша** рядом с пользователем
2. Измените нужные поля
3. Для смены пароля введите новый (оставьте пустым — не менять)
4. ⚠️ Если админ меняет **свой** пароль — нужно указать текущий пароль
5. Нажмите **"Сохранить"**

### 5.4. Блокировка пользователя

1. Нажмите иконку **замка** рядом с пользователем
2. Пользователь не сможет войти в систему, но останется в списке
3. Для разблокировки — нажмите иконку ещё раз

### 5.5. Удаление пользователя

1. Нажмите иконку **корзины**
2. Подтвердите удаление

⚠️ **Ограничения:**
- Нельзя удалить самого себя
- Нельзя удалить последнего администратора
- Нельзя заблокировать самого себя
- Нельзя понизить свою роль (чтобы не остаться без админа)

## 6. Разделы мониторинга

### 6.1. Дашборд

Главный экран со сводкой смены. Содержит:

- **5 KPI-карточек**: Выручка, Чеков, Средний чек, Активные столы, Блюд продано
- **График по часам** — динамика выручки и количества чеков
- **Круговая диаграмма** — структура типов оплат
- **Топ-10 официантов** с прогресс-барами
- **Топ-10 блюд** — bar-chart
- **Таблица последних чеков** — кликабельная, открывает детали

### 6.2. Баланс

Системный балансовый отчёт по типам оплат:

- KPI: общая выручка, чеков, гостей, типов оплат
- Bar-chart по типам оплат
- Таблица с долями (в процентах и прогресс-баром)
- Отдельный блок для налогов и сборов

### 6.3. Выручка

Детализация выручки по валютам и способам оплат:

- Сравнительный bar-chart по способам оплат
- По каждой валюте — раскрывающаяся карточка с методами

### 6.4. Открытые суммы

Активные (незакрытые) заказы по официантам:

- KPI: сумма, столы, гости, официанты
- Таблица: официант → количество столов/гостей → сумма

### 6.5. Чеки

Список закрытых чеков с детализацией:

- Поиск по официанту/кассиру/станции/номеру
- Клик по чеку → детальный просмотр:
  - Список позиций с кодом, названием, количеством, ценой
  - Применённые скидки
  - Итоговая сумма

### 6.6. Заказы

Распределение активных столов по официантам:

- Карточки по каждому официанту
- Список номеров столов

### 6.7. Расход блюд

Отчёт о проданных блюдах:

- Топ-10 блюд bar-chart
- Полная таблица с поиском по названию/коду
- Сортировка: по сумме / количеству / названию

### 6.8. Официанты / Кассиры / Станции

Три аналогичных раздела с выручкой по персонам/станциям:

- KPI: общая выручка, активных, лучший результат, средняя выручка
- Топ-10 bar-chart
- Таблица с раскрывающимися деталями (клик по строке):
  - По каждой валюте — методы оплат с суммами

### 6.9. Персонал

Справочник сотрудников:

- Поиск по имени или коду
- Карточки с аватарами (инициалы)

### 6.10. Планы залов

Интерактивная карта столов:

- Сетка карточек залов → клик для входа
- SVG-визуализация зала с цветовой индикацией:
  - 🟢 Зелёный (primary) — стол занят (есть заказ)
  - ⚪ Серый (secondary) — стол свободен
  - 🟡 Жёлтая рамка (accent) — выбранный стол
- На каждом столе: номер, количество гостей, сумма заказа
- Список столов справа — клик по столу показывает детали

### 6.11. Кассовая дата

Текущая кассовая дата, время и период смены:

- 3 большие карточки с параметрами
- Визуальный индикатор периода (Утро / Вечер / День)

### 6.12. Сервис-печать

Конфигурация принтеров и сервисных сообщений:

- Список принтеров с их назначением
- История сервисных сообщений по каждому принтеру

### 6.13. Сообщения

Отправка сообщений персоналу:

- Выбор получателя из справочника (с поиском)
- Текст сообщения (до 500 символов)
- История отправленных с timestamp

## 7. Темы оформления и горячие клавиши

### Темы

В правом верхнем углу нажмите иконку солнца/луны для переключения:

- 🌙 **Тёмная тема** (по умолчанию) — глубокий уголь + изумрудный неон
- ☀️ **Светлая тема** — тёплая кремовая + изумрудный акцент

### Адаптивность

- **Десктоп** (≥1024px): сайдбар слева, основной контент справа
- **Планшет** (768–1023px): сайдбар сворачивается, открывается по кнопке меню
- **Телефон** (<768px): мобильная вёрстка с slide-out меню

### Кнопка "Обновить данные"

Иконка круговых стрелок в топбаре — принудительно перезагружает все данные с сервера.

## 8. Бэкап и восстановление

### Что нужно бэкапить

| Файл | Что содержит |
|------|-------------|
| `C:\Program Files\RK Web Monitor\data\rkwebmon.db` | База данных: серверы, пользователи, сообщения, сессии |

Вся конфигурация хранится в одном файле SQLite — `rkwebmon.db`.

### Создание бэкапа

```cmd
:: Остановите сервер
"C:\Program Files\RK Web Monitor\stop.bat"

:: Скопируйте файл БД
copy "C:\Program Files\RK Web Monitor\data\rkwebmon.db" "D:\Backup\rkwebmon-%date%.db"

:: Запустите сервер
"C:\Program Files\RK Web Monitor\start.bat"
```

### Восстановление из бэкапа

```cmd
:: Остановите сервер
"C:\Program Files\RK Web Monitor\stop.bat"

:: Замените файл БД
copy "D:\Backup\rkwebmon-2024-01-15.db" "C:\Program Files\RK Web Monitor\data\rkwebmon.db" /Y

:: Запустите сервер
"C:\Program Files\RK Web Monitor\start.bat"
```

### Автоматический бэкап (опционально)

Создайте `backup.bat` в планировщике Windows:

```cmd
@echo off
set BACKUP_DIR=D:\Backup\RKWebMonitor
if not exist "%BACKUP_DIR%" mkdir "%BACKUP_DIR%"
copy "C:\Program Files\RK Web Monitor\data\rkwebmon.db" "%BACKUP_DIR%\rkwebmon-%date%.db"
:: Удалять бэкапы старше 30 дней
forfiles /p "%BACKUP_DIR%" /m "*.db" /d -30 /c "cmd /c del @file"
```

## 9. Устранение неполадок

### Приложение не открывается в браузере

1. Проверьте, что консольное окно запущено (меню Пуск → RK Web Monitor)
2. Откройте `http://localhost:8083` вручную
3. Проверьте порт: `netstat -an | findstr :8083` — должен быть LISTENING
4. Посмотрите логи: `C:\Program Files\RK Web Monitor\logs\server.log`

### Порт 8083 занят

Измените порт через веб-интерфейс:

1. Войдите как admin
2. Настройки → вкладка "Система" → поле "Порт" → введите свободный порт (например, 8084)
3. Нажмите "Сохранить настройки"
4. Нажмите "Перезапустить службу" (или запустите `restart.bat`)

Альтернатива — вручную изменить `C:\Program Files\RK Web Monitor\app\.env`:

```
PORT=8084
```

После этого перезапустите службу:
```cmd
"C:\Program Files\RK Web Monitor\restart.bat"
```

⚠️ Если сменили порт — обновите правило Windows Firewall:
```cmd
netsh advfirewall firewall delete rule name="RK Web Monitor"
netsh advfirewall firewall add rule name="RK Web Monitor" dir=in action=allow protocol=TCP localport=8084
```

### Не могу войти (admin/admin не работает)

1. Остановите сервер (`stop.bat`)
2. Удалите файл `data\rkwebmon.db`
3. Запустите сервер — будет создана новая БД с admin/admin
4. ⚠️ Все пользователи и настройки будут потеряны!

### Не подключается к RK7 TCP

1. Проверьте адрес: `ping 192.168.1.10`
2. Проверьте порт: `telnet 192.168.1.10 15551`
3. Убедитесь, что firewall не блокирует порт на сервере RK7
4. Проверьте логин/пароль — это учётка R-Keeper, а не Windows
5. Проверьте cryptKey (если используется)

### Не приходят сообщения персоналу

1. Убедитесь, что получатель активен в справочнике
2. Проверьте в Настройках → Персонал, что код сотрудника правильный
3. Сообщения отправляются на станцию RK7 — должна быть подключена к серверу

### Сервер упал и не поднимается

```cmd
:: Полный сброс
"C:\Program Files\RK Web Monitor\stop.bat"
taskkill /f /im node.exe
del "C:\Program Files\RK Web Monitor\app\.next\cache" /s /q /f
"C:\Program Files\RK Web Monitor\start.bat"
```

### Демо-данные пропали

Переустановите приложение или скопируйте папку `public\demo-data\` из репозитория GitHub.

## 10. Часто задаваемые вопросы

### Можно ли запустить на Linux/macOS?

Да. Исходный код кроссплатформенный. Запуск:
```bash
git clone https://github.com/AVzhirov/webmon
cd webmon
bun install
bun run db:push
bun run dev
```

Но Windows-установщик работает только на Windows.

### Можно ли работать с несколькими серверами RK7 одновременно?

Да. Добавьте все серверы в Настройках. На экране авторизации выбирайте нужный. Однако одновременно мониторится только один выбранный сервер.

### Безопасны ли пароли?

- Пароли пользователей хранятся в БД в виде scrypt-хеша (Node.js crypto, salt=16 bytes, key=64 bytes)
- Пароли RK7 хранятся в открытом виде (это необходимо для подключения) — обеспечьте физическую защиту БД
- Сессии — 7 дней, cookie с HttpOnly + SameSite=Lax

### Как изменить порт?

В файле `.env`:
```
PORT=8080
```
Перезапустите сервер. Обновите правило Firewall.

### Можно ли поставить как Windows-службу?

Да, через NSSM (входит в установку):

```cmd
:: Установка службы
"C:\Program Files\RK Web Monitor\bin\nssm.exe" install RKWebMonitor "C:\Program Files\RK Web Monitor\node\node.exe" "C:\Program Files\RK Web Monitor\app\server.js"

:: Автозапуск
"C:\Program Files\RK Web Monitor\bin\nssm.exe" set RKWebMonitor Start SERVICE_AUTO_START

:: Запуск
"C:\Program Files\RK Web Monitor\bin\nssm.exe" start RKWebMonitor
```

### Как обновить приложение?

1. Сделайте бэкап `data\rkwebmon.db`
2. Деинсталлируйте старую версию (данные сохранятся)
3. Установите новую версию
4. БД совместима между версиями (Prisma миграции)

### Где хранятся логи?

- Серверные: `C:\Program Files\RK Web Monitor\logs\server.log`
- Браузерные: DevTools (F12) → Console

### Можно ли ограничить доступ по IP?

Используйте Windows Firewall:
1. Удалите общее правило для порта 8083
2. Добавьте правило, разрешающее только с определённых IP

### Поддерживается ли HTTPS?

Из коробки — нет (HTTP на localhost). Для production рекомендуется:
1. Поставить Caddy или Nginx как reverse-proxy
2. Настроить Let's Encrypt сертификат
3. Проксировать на localhost:8083
